FoCS Tick 3*: Sets as Binary Search Trees

Required submission: tick3star.ml

A binary search tree uses an ordering on the labels to represent a dictionary of (key, value) pairs. But if we eliminate the stored values and have only keys as labels, then a binary search tree represents a set of keys. The tasks below concern this representation of sets of strings. There is no need to balance the trees you produce.

  1. Code functions insert(a,t) and member(a,t) to insert a string into a set and to test whether a given string belongs to a set.

Required functions:

val insert = fn : string * string tree -> string tree
val member = fn : string * string tree -> bool
  1. Write an ML function union(t1,t2) to return the union of two given binary search trees, namely the tree consisting of all of the elements of t1 and t2.

Required functions:

val union = fn : string tree * string tree -> string tree
  1. Write an ML function inter(t1,t2) to return the intersection of two given binary search trees, namely the tree consisting of all of the elements in common to t1 and t2.

Required functions:

val inter = fn : string tree * string tree -> string tree
  1. Code the function remove(a,t) to remove an element from a set.

Required functions:

val remove = fn : string * string tree -> string tree
